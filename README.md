## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 레지스트리 등록 및 메뉴 일부 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 리스트 안에 리스트를 저장하여 많은 자료형을 효율적으로 저장하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 사용자가 옳지 않은 형식의 입력을 받는것에 대비하고 알맞은 입력에 따라 가치를 출력하는 로직을 목표로 합니다.
  * input()으로 물건의 키, 이름, 가치,가 담긴 리스트를 입력받아 리스트에 저장.
  * if-elif문을 사용하며 이미 같은 이름의 물건/재료가 있는지 판별
  * 조건문과 리스트를 사용하여 잘못된 키 값을 받았을 경우 다시 키 값을 부여함  
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1:  로직 개선점 도출**
    * **프롬프트 요약:** "이게 내가 지금까지 짠 로직인데 개선할 점이 있다면 알려줘"
    * **적용 내용:** AI가 제안한 코드 입력 시 리스트에 없는 코드를 입력했을 경우에 대한 예외 처리에 대한 부분을 코드로 추가함
2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "어떻게 하면 현재 코드에서 사용되는 리스트를 효율적으로 사용할 수 있는지 알려줘!
    * **적용 내용:** 단순 1차원 리스트를 넘어, 리스트안에 리스트를 저장하는 구현이 매우 쉽고 직관적방법을 알려주어 코드에 적용함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 레지스트리 리스트를 추가했는데 키 값이 뒤죽박죽으로 구현된 현상**
    * **원인:** `리스트의 키값은 append 함수를 사용하여 리스트를 추가하면 1씩 올랐는데, 리스트 삭제를 마지막 인덱스가 아닌 다른 인덱스를 삭제하여 비는 현상
    * **해결:** `순서대로 리스트의 인덱스가 생성되도록 잘못된 인덱스를 지우고 다시 순서대로 번호를 부여
2. **문제 2: 조건문이 실행이 되지 않는 문제**
    * **원인:** 다른 조건문과 범위가 중첩되어 코드가 실행이 되지 않음.
    * **해결:** 파이썬의 모든 조건문의 들여쓰기는 범위를 표현함을 유의하고 올바르게 수정하여 학습함.
